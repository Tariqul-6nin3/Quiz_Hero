Q.1: Write down the difference among Map, Filter, ForEach and Find?
Main difference among those methods of javascript:

  • Map: This method creates a new array with the results of calling a function on every element in the array. It returns a new array with the same length as the original array.
  • Filter: This method creates a new array with all the elements that pass the test implemented by the provided function. It returns a new array with only the elements that meet the specified condition.
  • ForEach: This method calls a function for each element in the array. It does not return anything, it simply iterates over the array and performs the specified action on each element.
  • Find: This method returns the value of the first element in the array that satisfies the provided function. It returns undefined if no element satisfies the condition.

    • Q.2: Write down the difference among LocalStorage, session storage and Coockies?

    LocalStorage, sessionStorage, and cookies are all client-side storage options, but they have some key differences:

    Scope Difference:
  • LocalStorage and sessionStorage are scoped to the current domain.
  • Cookies can be set to a specific domain or to the entire domain tree.
    • Capacity Difference:
  • LocalStorage and sessionStorage have a larger storage capacity (up to 10 MB or more)
    compared to cookies (up to 4 KB).
    • Expiration Difference:
  • Cookies have an expiration date, while localStorage and sessionStorage do not.
  • This means that cookies can persist even after the user closes the browser, while LocalStorage and sessionStorage are cleared when the user closes the tab or window.
    • Accessibility Difference:
  • Cookies can be accessed by both the client-side (JavaScript) and server-side (PHP, Node.js, etc.) code.
  • LocalStorage and sessionStorage can only be accessed by the client-side code.
    • Q.3: What is Event Loop & how does it work?

    Definition of Event Loop: An event loop is a programming construct that manages the flow of events in a program. It operates by continuously checking for and processing events in a program's event queue, which are typically generated by external sources such as user input, network requests, or timers. The event loop retrieves events from the queue and executes the appropriate event handler functions to perform any necessary actions. This allows programs to handle complex event-driven scenarios in a structured and efficient manner.

    Here's a more detailed breakdown of how an event loop works:

  • The program creates an event queue, which is essentially a list of events that need to be processed.
  • The program registers event listeners that are responsible for handling specific types of events. For example, a button click might be handled by a "click" event listener.
  • When an event occurs, it is added to the event queue.
  • The event loop continuously checks the event queue for new events.
  • If an event is found in the queue, the event loop retrieves the event and sends it to the appropriate event listener.
  • The event listener processes the event and performs any necessary actions.
  • Once the event has been processed, the event listener returns control to the event loop, which continues to check the event queue for new events.
